# データ同期の仕様
## 仕様の方向性
双方向のデータ伝送をフレーム化して小さな単位で伝送する。

## 伝送データ形式：
JAVA(Android)仕様で記述する。JavaScript側は適宜オブジェクトまたはJSONを切り替えながら実装する。
### クラス
#### RVCloudSyncDataFrame 
データ伝送のフレームとなるクラス。これをJSON化したものをやり取りする。
##### プロパティ
- FrameCategory: enum 

    フレームのカテゴリを表す。

    - LOGIN_REQUEST
    - LOGIN_RESPONSE
    - CREATE_USER_REQUEST
    - CREATE_USER_RESPONSE
    - SYNC_DATA_REQUEST
    - SYNC_DATA_RESPONSE
    - DEVICE_DATA_FRAME
    - DEVICE_DATA_END_FRAME
    - CLOUD_DATA_FRAME
    - CLOUD_DATA_END_FRAME
- userName: String

    条件付き必須。ユーザ名、パスワードでリクエストするときは必須。authTokenでリクエストするときはnull許容。

- password: String

    条件付き必須。ユーザ名、パスワードでリクエストするときは必須。authTokenでリクエストするときは不要。

- dataBody: String

    データボディにはそれぞれオブジェクトをJSON文字列化したものが格納され、カテゴリに応じてクラスインスタンスに復号する。

- authToken: String
- lastSyncDate: long　最終同期日時ミリ秒 long型

    条件付き必須。データ同期リクエスト時は必要

- statusCode: enum

    レスポンス時に付加するステータスコード

    - STATUS_200_SYNC_START_OK
    - STATUS_200_SYNC_END_OK
    - STATUS_201_CREATED
    - STATUS_202_AUTHENTICATED
    - STATUS_400_DUPLICATE_USER_NAME
    - STATUS_400_SHORT_USER_NAME
    - STATUS_400_SHORT_PASSWORD
    - STATUS_400_WRONG_ARGS
    - STATUS_401_UNAUTHORIZED
    - STATUS_404_NOT_FOUND

## 処理の流れ
### ログインリクエスト（デバイス側から発信）
- frameCategory: LOGIN_REQUEST
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: null


### ログインレスポンス（クラウド側から発信）
#### ログイン成功時
- frameCategory: LOGIN_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: STATUS_202_AUTHENTICATED

    データ同期リクエストへ移行する。

#### ログイン失敗時
##### パスワードが違う
- frameCategory: LOGIN_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: STATUS_401_UNAUTHORIZED
##### ユーザ名が存在しない
- frameCategory: LOGIN_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: STATUS_404_NOT_FOUND

    ユーザの作成を促す。

### ユーザ作成リクエスト（デバイス側から発信）
- frameCategory: CREATE_USER_REQUEST
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: null

### ユーザ作成レスポンス（クラウド側から発信）
#### ユーザの作成に成功
- frameCategory: CREATE_USER_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: STATUS_201_CREATED
#### ユーザの作成に失敗
##### すでにそのユーザ名が存在する。
- frameCategory: CREATE_USER_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: null
- statusCode: STATUS_400_DUPLICATE_USER_NAME
##### ユーザ名またはパスワードが短い
デバイス側ではじいているの基本的には返されることはない。念のため、ステータスコードを実装する。
### データ同期リクエスト（デバイス側から発信）
#### ユーザ名とパスワードでリクエスト
- frameCategory: SYNC_DATA_REQUEST
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: 必須
- statusCode: null
#### authTokenでリクエスト
- frameCategory: SYNC_DATA_REQUEST
- userName: null
- password: null
- dataBody: 必須（更新されたtoken）
- authToken: 必須（DBに保存してあるtoken）
- lastSyncDate: 必須
- statusCode: null

    認証サーバが提供する情報に基づきユーザ名を生成する。

    優先度
    1. email_address
    2. first_name
    3. last_name
    
    いずれも空白の場合はnull許容
### データ同期レスポンス（クラウド側から発信）
#### ユーザ名とパスワードでリクエストした場合
##### 同期リクエスト成功時
- frameCategory: SYNC_DATA_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: 必須
- lastSyncDate: 必須
- statusCode: STATUS_200_SYNC_START_OK
##### 同期リクエスト失敗時 (パスワードが違う)
- frameCategory: SYNC_DATA_RESPONSE
- userName: 必須
- password: 必須
- dataBody: null
- authToken: null
- lastSyncDate: 必須
- statusCode: STATUS_401_UNAUTHORIZED
#### authTokenでリクエストした場合
##### 同期リクエスト成功時
- frameCategory: SYNC_DATA_RESPONSE
- userName: null許容
- password: null
- dataBody: null
- authToken: 必須
- lastSyncDate: 必須
- statusCode: STATUS_200_SYNC_START_OK
    authTokenリクエスト時は認証サーバにてすでに認可されている。そのアカウントが存在しない場合は新規作成するので必ず成功する。
##### 同期リクエストの失敗
ユーザ名、パスワード、authTokenの引数取り合わせがおかしいと失敗する。
- frameCategory: SYNC_DATA_RESPONSE
- userName: ?
- password: ?
- dataBody: null
- authToken: null
- lastSyncDate: 必須
- statusCode: STATUS_400_WRONG_ARGS
### デバイスデータ伝送フレーム（デバイス側から）
デバイス側に最終同期日時以降に蓄積したデータ件数分このフレームを送る
- frameCategory: DEVICE_DATA_FRAME
- userName: null
- password: null
- dataBody: 必須
- authToken: 必須
- lastSyncDate: null
- statusCode: null
### デバイスデータ伝送終了フレーム（デバイス側から）
デバイス側に蓄積したデータが伝送し終えたなら、このフレームを送る。このフレームを受けてクラウド側が伝送を開始する。
- frameCategory: DEVICE_DATA_END_FRAME
- userName: null
- password: null
- dataBody: null
- authToken: 必須
- lastSyncDate: null
- statusCode: null
### クラウドデータ伝送フレーム（クラウド側から）
クラウド側に最終同期日時以降に蓄積したデータ件数分このフレームを送る
- frameCategory: CLOUD_DATA_FRAME
- userName: null
- password: null
- dataBody: 必須
- authToken: 必須
- lastSyncDate: null
- statusCode: null
### クラウドデータ伝送終了フレーム（クラウド側から）
クラウド側に蓄積したデータが伝送し終えたなら、このフレームを送る。これをもってデータ同期を終了とする。
- frameCategory: CLOUD_DATA_END_FRAME
- userName: null
- password: null
- dataBody: null
- authToken: 必須
- lastSyncDate: null
- statusCode: null
